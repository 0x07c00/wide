Wide 的由来

  • Go IDE 很少，用 Go 本身开发的 IDE 更是没有，这是一个很好的尝试
  • 基于 Web 的团队 IDE：
    • 客户端只需要浏览器
    • 团队开发，降低协作复杂度
    • 跨平台，甚至在移动设备上
    • For the geeks
  • 目前较为流行的 Go IDE 都有一些缺陷或遗憾：
    • 文本编辑器类（vim/emacs/sublime/Atom 等）：对于新手门槛太高，搭建复杂
    • 插件类（goclipse 等）：需要原 IDE 支持，不够专业
    • LiteIDE：同时只能运行一个进程；界面不够 modern
    • 没有团队开发体验
  • 商业开源的一次尝试
